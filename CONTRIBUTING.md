# Contributing

Thank you for helping improve AGENTROPOLIS-ONTOLOGY.

## What to contribute

Contributions should strengthen the ontology as a portable public resource. Good contributions include:

- clarifying documentation
- improving schema quality and interoperability
- adding examples that remain vendor-neutral
- tightening security and governance guidance

## Contribution expectations

- keep changes focused and reviewable
- avoid embedding secrets, credentials, or personal data
- prefer explicit, machine-readable structures over narrative-only changes
- document security implications when modifying schemas or examples
- preserve portability across runtimes and vendors

## Pull requests

When opening a pull request:

1. explain the problem being solved
2. summarize the ontology objects affected
3. describe any compatibility impact
4. note any security, privacy, or governance considerations
