# Overview

AGENTROPOLIS-ONTOLOGY defines shared structures for representing governed AI ecosystems without binding users to a single product, provider, or orchestration stack.

## Core concepts

- **Districts** group governance boundaries, participants, and operating controls.
- **Agents** describe autonomous or semi-autonomous actors.
- **Skills** describe capabilities an agent may invoke.
- **Memory** describes retained knowledge with lifecycle and sensitivity controls.
- **Policies** define guardrails, obligations, and enforcement intent.
- **Workflows** define ordered activities and control points.
- **Events** define observable state changes and audit-relevant activity.
- **Permissions, registries, and audits** support operational governance around the core objects.

## Intended use

Use this repository to:

- bootstrap public governance models
- exchange structured metadata between tools
- document operating assumptions for multi-agent systems
- validate ontology documents with common schema tooling

## Non-goals

- shipping a proprietary runtime
- encoding vendor-specific product contracts
- granting brand, lore, or trademark reuse rights
- replacing implementation-layer security controls
